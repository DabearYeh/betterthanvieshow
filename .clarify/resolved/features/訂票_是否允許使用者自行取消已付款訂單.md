# 釐清問題

使用者是否可以在前台自行取消狀態為「已付款」的訂單（即退票）？

# 定位

Feature：訂票.feature
Rule：未明確定義「已付款」後的退票流程

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 允許，且訂單狀態轉為「已退款」（Refunded），釋放座位與票券 |
| B | 不允許自行退票，必須聯繫客服或臨櫃辦理 |
| C | 允許，但僅限於電影開演前 X 分鐘可自行退票 |
| D | 完全不允許退票 |
| Short | Format: Short answer (<=5 words) |

# 影響範圍

- Entity: Order (需新增 Refunded 狀態), Ticket (需由 Used/Unused 轉回 Expired 或其他狀態)
- Feature: 訂票 (退票流程), 查詢訂票記錄 (顯示退票狀態)

# 優先級

High

---
# 解決記錄

- **回答**：目前沒有退票功能（已付款訂單無法取消）
- **更新的規格檔**：spec/erm.dbml
- **變更內容**：更新 Order.status 的 Note，註明「目前系統無退票功能，訂單一旦付款即為最終狀態」
