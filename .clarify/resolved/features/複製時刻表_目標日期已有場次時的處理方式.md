# 釐清問題

當目標日期 (Target Date) 的狀態為 Draft，但該日期下「已經存在部分手動建立的場次」時，執行「複製時刻表」操作應如何處理？

# 定位

Feature：複製時刻表.feature
Rule：只能複製到草稿(Draft)狀態的日期

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 覆蓋模式：清除目標日期現有的所有場次，完全替換為來源日期的場次 |
| B | 合併模式：保留現有場次，將來源日期的場次追加 (Append) 到目標日期（需檢查時間衝突） |
| C | 衝突報錯：若目標日期已有任何場次，禁止執行複製，需先手動清空 |
| Short | Format: Short answer (<=5 words) |

# 影響範圍

- Feature: 複製時刻表
- Rule: 複製邏輯與資料寫入策略

# 優先級

Medium

---
# 解決記錄

- **回答**：A - 覆蓋模式：清除目標日期現有的所有場次，完全替換為來源日期的場次
- **更新的規格檔**：spec/features/複製時刻表.feature
- **變更內容**：更新 Rule，明確規範複製行為會先清除目標日期的所有舊場次
