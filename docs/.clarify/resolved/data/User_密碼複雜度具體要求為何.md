# 釐清問題

密碼複雜度要求的具體規則是什麼？（如字元長度、大小寫、數字、特殊字元的要求組合）

# 定位

ERM：User.password 屬性
Feature：使用者註冊 → Rule: 密碼必須符合複雜度要求 → Example: 密碼過於簡單時提示

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 最少 8 字元，包含大小寫字母與數字 |
| B | 最少 8 字元，包含大小寫字母、數字與特殊字元 |
| C | 最少 6 字元，包含大小寫字母與數字 |
| D | 最少 10 字元，包含大小寫字母、數字與特殊字元 |
| Short | 其他具體規則 |

# 影響範圍

- User 實體的密碼驗證邏輯
- 使用者註冊功能的前置驗證
- API 密碼變更相關功能

# 優先級

High

---

**說明**：Feature 中已提到「密碼至少需 8 字元，包含大小寫字母與數字」，但需確認是否還需要特殊字元，以及是否需要其他限制條件。

---
# 解決記錄

- **回答**：A - 最少 8 字元，包含大小寫字母與數字
- **更新的規格檔**：無需修改（當前設計已符合需求）
- **變更內容**：
  - 決策：密碼複雜度要求為「最少 8 字元，包含大小寫字母與數字」
  - 具體規則：
    - 長度：≥ 8 字元
    - 必須包含：大寫字母（A-Z）
    - 必須包含：小寫字母（a-z）
    - 必須包含：數字（0-9）
    - 不強制要求：特殊字元
  - 當前 spec/features/使用者註冊.feature 中的提示訊息「密碼至少需 8 字元，包含大小寫字母與數字」已符合此決策
