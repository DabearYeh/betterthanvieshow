# 釐清問題

已販售 (OnSale) 狀態的時刻表是否可以在特殊情況下緊急取消販售？

# 定位

ERM：DailySchedule.status 狀態轉換規則
Feature：開始販售時刻表.feature - Rule: 販售狀態不可逆轉

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 絕對不可逆：OnSale 後無論任何情況都無法改變，需透過其他方式處理（如建立新日期） |
| B | 緊急取消：特殊情況下（如電影臨時取消）可由超級管理者將狀態改回 Draft |
| C | 暫停販售：新增 Suspended 狀態，可暫停販售但不可編輯場次 |
| Short | 提供其他簡短答案（<=5 字）|

# 影響範圍

- DailySchedule 狀態機設計
- 開始販售時刻表功能的可逆性
- 緊急情況處理流程
- 已售票券的處理（若狀態可逆轉）

# 優先級

Medium
- 影響緊急情況處理能力，但正常情況不需要

---
# 解決記錄

- **回答**：A - 絕對不可逆：OnSale 後無論任何情況都無法改變，需透過其他方式處理
- **更新的規格檔**：spec/erm.dbml
- **變更內容**：強化 DailySchedule.status 的 note，明確狀態轉換絕對不可逆（無例外）
