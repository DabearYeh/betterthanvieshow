# 釐清問題

電影時長屬性是否有最大值限制？如果有，上限是多少？

# 定位

ERM：Movie.duration 屬性
Feature：建立電影、編輯電影 → Rule: 時長必須大於 0

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 無上限，只要求大於 0 |
| B | 最多 480 分鐘（8 小時） |
| C | 最多 360 分鐘（6 小時） |
| D | 最多 300 分鐘（5 小時） |
| Short | 其他上限值 |

# 影響範圍

- Movie 資料驗證規則
- 建立/編輯電影的 API 驗證邏輯
- 場次衝突檢查的計算邏輯

# 優先級

Medium

---

**說明**：目前規格只定義了下限（必須 > 0），未定義上限。需要確認是否有合理的電影時長上限。

- **回答**：A - 無上限，只要求大於 0
- **更新的規格檔**：無需修改（當前設計已符合需求）
- **變更內容**：
  - 決策：Movie.duration 無上限限制，只要求 > 0
  - 當前 erm.dbml 中 Movie.duration 定義為「時長（分鐘），必須 > 0」，符合此決策
