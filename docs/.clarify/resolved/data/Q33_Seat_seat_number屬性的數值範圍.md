# Q33: Seat_seat_number 屬性的數值範圍

## 問題
Seat.seat_number 屬性的數值範圍為何？

## 背景
影廳的座位編號通常從 1 開始連續編號，但可能因柱子、通道等原因出現空隙。

## 選項
- **A**：正整數，一般從 1 開始，中間可能有空隙（需用 is_active 標記無效座位）
- **B**：正整數，必須連續無空隙
- **C**：可以自訂，允許字母數字混合

## 使用者選擇
**A** - 正整數，一般從 1 開始，若因空間限制無法使用則設 is_active=false

## 理由
支援實際業務需求：
- 影廳中常因柱子、走道等空間限制導致某些「座位號碼」無法使用
- 使用 is_active 機制優雅地標記這類座位，避免座位號碼跳號
- 使用者可直觀瞭解影廳實際座位位置和編號

## 影響的規格檔案
- `spec/erm.dbml`: Seat 表更新
  - seat_number: int [note: '座位號碼(正整數,一般從1開始)']
  - is_active: bool [note: '座位是否有效,系統預設有效。若因柱子、空間等原因無法使用,設為無效']

## 規格變更
1. Seat.seat_number 說明：正整數，一般從 1 開始
2. Seat.is_active 說明：增加「若因柱子、空間等原因無法使用,設為無效」的說明
3. 確認 (row_name, seat_number) 組合唯一約束（與 Q16 相同）
