# Q34: Movie_poster_url 屬性是否為必填欄位

## 問題
Movie.poster_url 屬性是否為必填欄位？

## 背景
電影海報 URL 用於在前台系統顯示電影封面圖片。但可能存在某些電影暫無海報的情況。

## 選項
- **A**：必填欄位，建立電影時一定要上傳海報
- **B**：非必填欄位，允許暫無海報
- **C**：必填欄位，但系統提供預設海報

## 使用者選擇
**B** - 非必填欄位

## 理由
- 允許系統更靈活：新電影資訊可能先建檔，海報稍後上傳
- 減少建立電影時的必填項限制
- 前台可使用預設圖片或播放狀態提示代替

## 影響的規格檔案
- `spec/erm.dbml`: Movie 表更新
  - poster_url: string? [note: '電影海報URL']

## 規格變更
1. Movie.poster_url 從必填改為非必填（可 null）
