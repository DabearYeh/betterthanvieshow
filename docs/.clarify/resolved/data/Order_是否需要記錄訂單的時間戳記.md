# 釐清問題

Order 是否需要記錄訂單的時間戳記（如建立時間、付款時間）？

# 定位

ERM：Order 實體，可能需要新增 created_at、paid_at 等時間戳記屬性

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 需要建立時間與付款時間 |
| B | 僅需要建立時間 |
| C | 需要建立時間、付款時間、更新時間 |
| D | 不需要時間戳記 |
| Short | 提供其他簡短答案（<=5 字）|

# 影響範圍

- Order 實體的屬性定義
- 訂票功能的後置條件
- 查詢訂票記錄功能的資料呈現與排序
- 訂單過期判斷邏輯（若有付款時限）

# 優先級

Medium
- 影響訂單追蹤與記錄功能
- 影響付款時限功能（若需實作）

---

# 解決記錄

**使用者選擇：C**

**規格文件變更：**
- `spec/erm.dbml` - Order 實體新增以下屬性：
  - `created_at string [note: '訂單建立時間']`
  - `paid_at string [note: '付款完成時間']`
  - `updated_at string [note: '訂單更新時間']`
