# 釐清問題

Order 實體的 order_status 應包含哪些狀態？狀態之間的轉換規則如何？

# 定位

ERM：Order 實體 order_status 屬性
Feature：訂票.feature

# 問題詳情

當前 erm.dbml 定義：`order_status string [note: '訂單狀態（待付款、已付款、已完成、已取消、已退款）']`

但訂票流程中只涉及：待付款 → 已付款 → 已完成

需要釐清：
- 什麼情況下會轉移到已取消或已退款？
- 是否還有其他狀態？
- 狀態轉換是否有時間限制？

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 待付款 → 已付款 → 已完成，另有已取消（訂單刪除）、已退款（退票） |
| B | 完整列出 5 個狀態的轉換規則：哪些狀態可轉移到哪些狀態 |
| C | 其他（請說明） |

# 影響範圍

- Order 狀態管理
- 訂票.feature 與後續流程的規則
- 資料庫約束與業務邏輯

# 優先級

Medium - 影響完整的業務流程
