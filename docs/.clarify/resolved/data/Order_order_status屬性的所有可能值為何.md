# 釐清問題

Order 的 order_status 屬性的所有可能值為何？

# 定位

ERM：Order 實體的 order_status 屬性

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 待付款、已付款、已取消 |
| B | 待付款、已付款、已完成、已取消、已退款 |
| C | 已建立、已付款、已出票、已取消 |
| Short | 提供其他簡短答案（<=5 字）|

# 影響範圍

- Order 實體的屬性定義與狀態轉換規則
- 訂票功能的後置條件
- 查詢訂票記錄功能的資料呈現
- 訂單生命週期管理

# 優先級

High
- 影響訂單狀態的核心定義
- 影響訂票流程的業務邏輯
