# 釐清問題

Ticket 是否需要記錄票價？

# 定位

ERM：Ticket 實體，可能需要新增 price 屬性

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 需要，每張票券記錄其票價 |
| B | 不需要，票價由場次或座位類型決定 |
| C | 需要，且需記錄原價與折扣後價格 |
| Short | 提供其他簡短答案（<=5 字）|

# 影響範圍

- Ticket 實體的屬性定義
- Order 總金額的計算方式
- 訂票功能的後置條件
- 票價管理邏輯（場次定價、座位定價）

# 優先級

High
- 影響票價管理的核心資料結構
- 影響訂單金額計算邏輯

---
# 解決記錄

- **回答**：C - 需要，且需記錄原價與折扣後價格
- **更新的規格檔**：spec/erm.dbml
- **變更內容**：在 Ticket 實體中新增 original_price（票券原價）、final_price（折扣後價格）兩個屬性
