# 釐清問題

ERM 中提到「目前只有一個環境，但架構需允許未來多影城擴充」。這個「多影城」功能的具體需求是什麼？當前版本是否需要實現？

# 定位

ERM：Cinema 實體與 Theater 的關聯關係

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 當前版本僅支援單影城，未來預留多影城架構 |
| B | 當前版本需實現多影城支援，但 UI 可簡化 |
| C | 當前版本支援多影城，完整 UI 與管理功能 |
| Short | 其他多影城需求說明 |

# 影響範圍

- Cinema 實體的實現範圍
- 管理功能（建立/編輯/刪除影廳）的影城選擇邏輯
- API 設計中的影城篩選
- 前台使用者的影城選擇流程

# 優先級

High

---

**說明**：決定多影城支援的優先級和實現範圍對於系統架構設計有重大影響。

---
# 解決記錄

- **回答**：A - 當前版本僅支援單影城
- **更新的規格檔**：無需修改（當前設計已符合需求）
- **變更內容**：
  - 決策：當前版本僅支援單影城，不需要 Cinema 實體
  - Theater 直接管理，無需上層 Cinema 實體
  - 簡化架構，快速上市
  - 未來如需多影城擴充，可在後續版本中新增 Cinema 實體和 Cinema 1:N Theater 關聯
  - 當前 erm.dbml 中沒有 Cinema 實體，符合此決策
