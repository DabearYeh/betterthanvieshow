# 釐清問題

Seat 的 is_active 屬性何時會變更為無效？

# 定位

ERM：Seat 實體的 is_active 屬性生命週期管理

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 座位損壞或維修時設為無效 |
| B | 管理者手動設定 |
| C | 系統不會變更，永遠有效 |
| Short | 提供其他簡短答案（<=5 字）|

# 影響範圍

- Seat 實體的狀態管理
- 設定座位配置功能的規則
- 訂票功能的座位可用性判斷
- 是否需要座位停用/啟用功能

# 優先級

Medium
- 影響座位管理邏輯
- 影響訂票時的座位篩選

---

# 解決記錄

- **回答**：C - 系統不會變更，永遠有效（座位配置一旦建立就固定，無維修/停用功能）
- **更新的規格檔**：spec/erm.dbml
- **變更內容**：Seat 實體 is_active 屬性預設為有效，系統設計中不提供後續變更此屬性的功能；若座位需要停用，應在建立時就標記為走道而非座位
