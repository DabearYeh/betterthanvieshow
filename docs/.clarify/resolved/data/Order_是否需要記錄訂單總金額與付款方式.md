# 釐清問題

Order 是否需要記錄訂單總金額與付款方式？

# 定位

ERM：Order 實體，可能需要新增 total_amount、payment_method 等屬性

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 需要總金額與付款方式 |
| B | 僅需要總金額，付款方式由 Ticket 記錄 |
| C | 需要總金額、付款方式、交易編號等 |
| D | 不需要，由 Ticket 的票價加總計算 |
| Short | 提供其他簡短答案（<=5 字）|

# 影響範圍

- Order 實體的屬性定義
- 訂票功能的後置條件
- 查詢訂票記錄功能的資料呈現
- 付款流程設計

# 優先級

High
- 影響訂單與付款的核心資料結構
- 影響訂票功能的完整性

---
# 解決記錄

- **回答**：C - 需要總金額、付款方式、交易編號等
- **更新的規格檔**：spec/erm.dbml
- **變更內容**：在 Order 實體中新增 total_amount（訂單總金額）、payment_method（付款方式）、transaction_id（交易編號）三個屬性
