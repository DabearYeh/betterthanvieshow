# 釐清問題

建立影廳時，座位配置方式為何？是否需要改為「選排數和列數，逐格標記」？

# 定位

ERM：Theater 實體、Seat 實體的座位配置流程

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 保持現有：輸入總座位數，逐座設定排號和座位號 |
| B | 改為：先選排數和列數，系統生成網格，逐格標記為「座位」或「走道」或「殘障座」 |
| C | 先選排數和列數，每個格子自訂為具體座位類型（一般座、VIP、殘障座、走道） |
| D | 其他（請說明） |

# 影響範圍

- Theater 實體是否需要新增 row_count、column_count 屬性
- Seat 實體的 is_active 是否改為 seat_category（座位類型）
- 設定座位配置功能的流程完全改變
- 前端座位配置繪製的 UI 邏輯

# 優先級

High
- 影響座位配置的核心流程
- 影響資料模型結構

---
# 【決議】

**選擇：C - 網格加類型定制方式**

## 依據

- 先選排數和列數，系統生成網格（便於視覺化管理）
- 每個格子可自訂為具體座位類型：
  - 一般座
  - VIP 座
  - 殘障座
  - 雙人座
  - 走道
  - 其他自訂類型
- 提供最大的靈活性和可擴展性

## 規格更新

- Theater 實體：新增 row_count、column_count 屬性
- Seat 實體：保留 is_active（標記座位是否有效），seat_type 用來記錄具體類型
- 設定座位配置.feature：更新流程為網格選擇  逐格選擇類型
- 座位配置會影響 Theater.total_seats 的值（必須同步）
