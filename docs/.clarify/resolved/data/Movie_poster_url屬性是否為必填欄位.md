# 釐清問題

Movie 的 poster_url 屬性是否為必填欄位？

# 定位

ERM：Movie 實體的 poster_url 屬性

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 必填（所有電影都必須有海報） |
| B | 非必填（電影可以沒有海報） |
| Short | 提供其他簡短答案（<=5 字）|

# 影響範圍

- Movie 實體的屬性定義
- 建立電影功能的前置條件
- 編輯電影功能的驗證規則
- 瀏覽電影功能的顯示邏輯

# 優先級

Medium
- 影響電影建立與編輯的驗證邏輯
- 影響前台顯示邏輯

---

# 解決記錄

- **回答**：B - 非必填，電影可以沒有海報
- **更新的規格檔**：spec/erm.dbml
- **變更內容**：Movie 實體 poster_url 屬性已定義為非必填，無需修改
