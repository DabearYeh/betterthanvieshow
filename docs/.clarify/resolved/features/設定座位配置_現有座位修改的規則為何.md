# 設定座位配置_現有座位修改的規則為何

設定座位配置時,如果座位已被訂購,管理者是否可以修改其座位類型或是否有效（is_active）？

# 定義

Feature：設定座位配置
ERM：Seat.seat_type, Seat.is_active 的修改限制

# 多選項

| 選項 | 描述 |
|--------|-------------|
| A | 已訂購座位無法修改任何屬性 |
| B | 可修改座位類型,但不可改變 is_active（無法停用已訂購座位）|
| C | 完全禁止修改已訂購座位 |
| D | 可修改任何屬性,系統自動處理衝突 |
| Short | 其他修改規則 |

# 影響範圍

- 座位配置編輯功能的設計
- Seat 實體的修改限制控制
- 座位類型變更的業務邏輯

# 優先級

Medium

---

**說明**：規格未定義已訂購座位的座位配置修改限制,這影響座位管理的穩定性與安全性。

---

# 解決記錄

- **決議**：B  可訂購後確認修改,保留已訂購座位
- **更新規格檔案**：spec/features/設定座位配置.feature
