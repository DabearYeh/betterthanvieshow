# 釐清問題

系統中使用者的角色有哪些？不同角色的權限邊界是什麼？（如一般使用者、影城管理者、驗票人員等）

# 定位

Feature：使用者登入、使用者註冊（隱含的角色系統）
涉及所有需要權限控制的功能

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 兩個角色：一般使用者（訂票）、管理者（管理電影、影廳、場次） |
| B | 三個角色：一般使用者、管理者、驗票人員 |
| C | 四個角色：一般使用者、管理者、驗票人員、超級管理員 |
| Short | 具體角色列表與權限矩陣 |

# 影響範圍

- User 實體是否需要 role 屬性
- 所有功能的權限檢查邏輯
- API 端點的訪問控制設計
- 登入後的功能導航
