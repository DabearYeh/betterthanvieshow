# 訂票_取消訂單的條件與流程是什麼

使用者在系統何時可以取消訂單？取消後票券狀態如何變更？座位如何釋放？

# 定義

Feature：訂票功能，規格中確認取消流程

# 多選項

| 選項 | 描述 |
|--------|-------------|
| A | 任何時候可取消；支付前在驗票前申請退款；驗票後無法取消 |
| B | 場次開始前任何時候可取消；場次開始後無法取消 |
| C | 付款後只有 24 小時內申請退款 |
| D | 付款後即無法取消 |
| Short | 其他取消規則 |

# 影響範圍

- Order 取消流程的條件與規則
- Ticket 狀態更新邏輯
- Seat 座位釋放機制
- 退款流程設計

# 優先級

High

---

**說明**：當前規格中提到 Order 有「已取消」與「已退款」兩種狀態,但未定義何時允許取消訂單及置條件,以及取消後的具體操作流程。

---

# 解決記錄

- **決議**：A  僅待付款前,可以取消
- **理由**：付款前可以取消,付款後的退款流程待後續實作或手動代理
- **更新規格檔案**：spec/features/訂票.feature
