# 設定座位配置_無效座位標記功能

設定座位配置.feature 描述座位可設定為有效（is_valid = true）,但是缺少「標記無效座位」的功能？ERM 中提到「若有柱子、空間限制而無法使用,設為無效」,但功能文件中缺少此操作流程。

# 定義

Feature：設定座位配置 feature

ERM：Seat 實體的 is_valid 屬性

# 多選項

| 選項 | 描述 |
|--------|-------------|
| A | 設定座位配置時支援標記無效座位（柱子遮擋、無法使用等）|
| B | 不在座位配置完成後,而在編輯影廳功能來標記無效座位 |
| C | 設定座位配置時每個座位可設定為「無效座位」標記為不可訂票功能 |
| D | 無效座位標記不是在系統功能中,而是人工手動更新資料庫 |

# 影響範圍

- 設定座位配置功能的完整性
- 編輯影廳功能的延伸功能
- 訂票時的座位過濾邏輯（排除無效座位）

# 優先級

Medium
- Medium：影響座位配置的完整性,但不影響主要訂票功能

---

# 解決記錄

- **決議**：A  設定座位配置時可以標記無效座位（柱子、無法使用等）
- **更新規格檔案**：無需修改（spec/features/設定座位配置.feature 已支援標記 Empty 類型）
- **變更內容**：現有規格已明確支援在初始配置時標記為「Empty」類型的座位（柱子等無法使用的位置）,無需額外修改需求

補充說明:
- 座位可訂購性判斷：「一般座位」與「殘障座位」可訂購 (is_valid = true)；「走道」與「Empty」不可訂購 (is_valid = false)
- Empty 的作用：標記為結構性障礙（如柱子）無法使用的位置
