# 驗票_驗票失敗的錯誤回饋

驗票.feature 中提到檢查都會有「操作失敗」（票券不存在、已使用、過期、場次已結束等）,但系統是否需要向驗票人員顯示具體的失敗原因？還是統一顯示什麼訊息？

# 定義

Feature：驗票 feature 中的操作失敗 Example

# 多選項

| 選項 | 描述 |
|--------|-------------|
| A | 系統顯示詳細的失敗原因,如「票券已使用」、「場次已結束」等,驗票人員可了解具體原因 |
| B | 系統統一顯示「票券無效」,不顯示失敗原因 |
| C | 系統記錄失敗原因至 TicketValidateLog.validation_result（欄位值）,但不顯示給操作者 |
| D | 系統顯示失敗且記錄至驗票日誌,並自動通知管理者 |

# 影響範圍

- 驗票系統的用戶體驗與錯誤訊息顯示設計
- TicketValidateLog 的資訊記錄完整性
- 驗票流程的操作便利性與診斷性

# 優先級

Low
- Low：不影響驗票邏輯本身,但影響客戶體驗與診斷

---

# 解決記錄

- **決議**：B  使用統一訊息,顯示原因碼
- **更新規格檔案**：spec/features/驗票.feature
