# 查詢訂票記錄_查詢功能的訪問權限為何

在「查詢訂票記錄」功能中,誰可以查詢哪些訂票記錄？是客戶查詢自己的訂單,還是管理者查詢所有訂單？

# 定義

Feature：查詢訂票記錄

# 多選項

| 選項 | 描述 |
|--------|-------------|
| A | 一般使用者只能查詢自己的訂票記錄；管理者可查詢所有使用者的記錄 |
| B | 所有登入使用者都能查詢所有訂票記錄 |
| C | 只管理者可查詢；一般使用者無法查詢 |
| D | 只訂單的建立人才能查詢 |
| Short | 其他訪問控制規則 |

# 影響範圍

- 查詢訂票記錄功能的權限檢查
- API 端點的訪問控制設計
- 資料隱私保護政策
- 報表與統計功能的使用者範圍

# 優先級

High

---

**說明**：Feature 中描述「影城管理者可以查詢訂票記錄」,但未明確是否一般使用者也可以查詢自己的訂單,以及功能具體的訪問權限。

---

# 解決記錄

- **決議**：A  嚴格隔離，一般使用者只能查詢自己的訂票記錄；管理者可查詢所有使用者的記錄
- **更新規格檔案**：無需修改（spec/features/查詢訂票記錄.feature 已符合此決議）
- **變更內容**：規格檔已包含此規則，無需進一步修改需求
