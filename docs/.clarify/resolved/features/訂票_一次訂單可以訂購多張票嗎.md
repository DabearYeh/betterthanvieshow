# 訂票_一次訂單可以訂購多張票嗎

訂票功能中,一次訂單可以訂購多張票（多個座位）嗎？

# 定義

Feature：訂票功能
Rule：Order 與 Ticket 的關係（Order 1:N Ticket）

# 多選項

| 選項 | 描述 |
|--------|-------------|
| A | 可以，一次訂單可訂多個座位 |
| B | 不可以，每次訂單只能訂一個座位 |
| Short | 其他（請簡短答案<=5 字）|

# 影響範圍

- Order 與 Ticket 的關係設計
- 訂票功能的流程規則
- 座位選擇的 UI 設計
- 測試案例設計

# 優先級

High
- 影響訂票功能的核心設計
- 影響 Order 與 Ticket 的關係定義

# 釐清記錄

**使用者選擇：A**

**規格文件變更：**
- `spec/features/訂票.feature` - 新增規則：
  - Rule「一次訂單可以訂購多張票」：明確一個 Order 可以包含多個 Ticket，即可選擇多個座位

---

# 解決記錄

- **決議**：B  可訂購多張票，最多 6 張，超過需分單
- **理由**：同一訂單內，使用者可以選擇 1-6 張票（座位需連續），超過需分單（使用者可以）
- **更新規格檔案**：spec/features/訂票.feature、spec/erm.dbml Order.ticket_count
