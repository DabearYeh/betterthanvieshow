# 設定場次_場次時間設定的最小粒度為何

設定場次時間輸入的最小單位是什麼？例如精確到分鐘？還是以 15 分鐘為單位？

# 定義

Feature：設定場次 Rule: 必須設定時段

# 多選項

| 選項 | 描述 |
|--------|-------------|
| A | 精確到分鐘（HH:MM）|
| B | 5 分鐘為單位 |
| C | 15 分鐘為單位 |
| D | 30 分鐘為單位 |
| Short | 其他粒度 |

# 影響範圍

- 場次時間設定的 UI 設計
- MovieShowTime.show_datetime 的儲存格式
- 場次衝突檢查的精確度
- 客戶體驗與操作輸入便利性

# 優先級

Low

---

**說明**：規格中未定義時間輸入的粒度,這會影響 UI 設計與時間衝突檢查的精確度。

---

# 解決記錄

- **決議**：A  精確到分鐘（HH:MM 格式開始時間）
- **更新規格檔案**：spec/erm.dbml MovieShowTime.start_time 欄位
