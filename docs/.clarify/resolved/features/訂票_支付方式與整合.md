# 訂票_支付方式與整合

訂票功能描述會進入付款流程,但付款方式並未定義。系統支援哪些支付方式？如何記錄付款資訊？

# 定義

Feature：訂票 feature 中「確認訂單後進入付款流程」規則

# 多選項

| 選項 | 描述 |
|--------|-------------|
| A | 信用卡（直接支付）|
| B | 第三方支付（如 LINE Pay、Apple Pay 等）|
| C | 模擬付款（測試用,自動通過）|
| D | A + B |
| E | A + B + C |
| Short | 其他付款方式 |

# 影響範圍

- 訂票流程的完整性
- 付款系統與第三方 API 的整合
- Order 表是否需要記錄支付方式的欄位紀錄
- 測試環境的支付模擬

# 優先級

Medium
- Medium：不影響功能規格本身,但影響付款實作與測試環境

---

# 解決記錄

- **決議**：B（變體）只支援第三方付款 LINE Pay
- **更新規格檔案**：spec/features/訂票.feature
- **變更內容**：補充「確認訂單後進入付款流程」規則,明確付款方式為 LINE Pay

補充說明:
- 付款方式：LINE Pay（第三方付款）
- 付款流程：訂單建立後導向 LINE Pay 付款頁面
- 付款記錄：付款成功後記錄於 Order.status 為「已付款」
- 測試模式：僅供模擬環境付款,待整合 LINE Pay API
