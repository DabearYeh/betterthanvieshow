# 釐清問題

開始販售時，是否需要檢查該日期至少有一個場次存在？

# 定位

Feature：開始販售時刻表.feature - Rule: 只有草稿狀態的時刻表可以開始販售

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 必須有場次：沒有任何場次的 DailySchedule 不可開始販售 |
| B | 可空販售：即使沒有場次也可開始販售（但實際上不會有票可賣） |
| C | 至少一廳：必須至少有一個影廳安排了場次才可販售 |
| Short | 提供其他簡短答案（<=5 字）|

# 影響範圍

- 開始販售時刻表功能的前置條件
- UI 的販售按鈕啟用條件
- 錯誤訊息提示

# 優先級

Medium
- 影響販售流程的完整性，但非核心功能阻礙

---
# 解決記錄

- **回答**：B - 可空販售：即使沒有場次也可開始販售
- **更新的規格檔**：spec/features/開始販售時刻表.feature
- **變更內容**：新增 Example「無場次也可開始販售」，明確允許空日期的時刻表開始販售
