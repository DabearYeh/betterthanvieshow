# 訂票_付款失敗時如何處理訂單與票券

訂票功能中,付款失敗時該如何處理已建立的訂單與票券？

# 定義

Feature：訂票 feature，關於「付款失敗時保留訂單並允許重新付款」

# 多選項

| 選項 | 描述 |
|--------|-------------|
| A | 立即刪除 Order 與相關 Ticket |
| B | 保留訂單並允許重新付款,座位保留一段時間 |
| C | 保留訂單但釋放座位,允許重新選擇 |

# 影響範圍

- 訂票.feature 中的付款失敗處理規則
- Order 與 Ticket 狀態管理
- 座位鎖定機制

# 優先級

High - 影響使用者體驗與業務邏輯

---

# 解決記錄

- **決議**：A  訂單保留,座位鎖定 3 分鐘後自動釋放
- **理由**：同一筆訂票訂單保留一致性,座位在 3 分鐘內被保留,逾時自動釋放給其他使用者
- **更新規格檔案**：spec/features/訂票.feature
