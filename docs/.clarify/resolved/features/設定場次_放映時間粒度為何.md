# 釐清問題

設定場次時，放映時間的最小粒度是多少？

# 定位

Feature：設定場次.feature
ERM：MovieShowTime.start_time 屬性

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 15 分鐘：放映時間必須是 15 分鐘的倍數（如 09:00, 09:15, 09:30） |
| B | 5 分鐘：放映時間必須是 5 分鐘的倍數 |
| C | 任意分鐘：放映時間可以是任意分鐘（如 09:07） |
| D | 30 分鐘：放映時間必須是 30 分鐘的倍數 |
| Short | 提供其他簡短答案（<=5 字）|

註：從 UI 圖片觀察，時間軸似乎以 15 分鐘為單位顯示。

# 影響範圍

- MovieShowTime.start_time 驗證規則
- 設定場次 UI 的時間選擇器設計
- 時間衝突檢查的精確度

# 優先級

Medium
- 影響 UI 設計與資料驗證，但不影響核心功能
