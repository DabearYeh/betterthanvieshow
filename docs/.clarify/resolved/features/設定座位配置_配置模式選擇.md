# 設定座位配置_配置模式選擇

座位配置功能的流程為何？是網格模式（輸入行列數）,還是逐座模式（逐個設定）？

# 定義

Feature：設定座位配置 > 座位配置流程

# 多選項

| 選項 | 描述 |
|--------|-------------|
| A | 保留現有：逐座設定,每個座位的座位編號、座位類型一一設定（一般座、VIP 等）|
| B | 改為：輸入行列數,系統產生 N×M 網格；每個格子可標記為「座位」「走道」「無效座位」|
| C | 改為：輸入行列數,系統產生網格,每個座位格子自訂類型（一般座、VIP、殘障座等）|
| D | 其他（請說明） |

- Seat 實體的結構與建立流程的改變
- 前端座位配置繪製的 UI 邏輯
