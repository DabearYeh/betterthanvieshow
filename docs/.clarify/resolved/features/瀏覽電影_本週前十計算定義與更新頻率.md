# 瀏覽電影_本週前十計算定義與更新頻率

「本週前十」的計算定義與更新頻率為何？

# 定義

Feature：瀏覽電影 > Rule: 可以查看本週前十電影

# 多選項

| 選項 | 描述 |
|--------|-------------|
| A | 週定義：週一到週日；銷售計數：已付款訂單；即時更新 |
| B | 週定義：過去 7 天；銷售計數：已付款訂單；每日更新 |
| C | 週定義：本週（週一到今天）；銷售計數：所有訂單；每小時更新 |
| D | 其他（請說明） |

# 影響範圍

- 瀏覽電影功能的計算邏輯
- 資料統計與快取策略
- 前台展示的即時性

# 解決記錄

- **決議**：B — 週定義：過去 7 天；銷售計數：已付款訂單；每日更新
- **理由**：過去 7 天的滾動窗口更符合「本週熱門」的概念；每日更新平衡即時性與效能
- **更新規格檔案**：spec/features/瀏覽電影.feature
- **補充說明**：
  - 過去 7 天定義：自然滾動,避免週一時清空
  - 已付款訂單：確保只計算真正完成的銷售
  - 每日更新：降低系統負擔與快取更新成本
