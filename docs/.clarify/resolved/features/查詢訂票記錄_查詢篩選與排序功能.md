# 釐清問題

查詢訂票記錄.feature 規定「系統列出訂單」，但未定義訂單的篩選與排序方式。例如：

- 訂單是否可按狀態篩選（未付款、已付款、已取消）？
- 訂單是否可按日期篩選或排序？
- 是否支持搜尋功能（如按訂單編號）？
- 是否分頁顯示？

# 定位

Feature：查詢訂票記錄.feature 的「列出訂單」規則

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 列出所有訂單，按建立日期倒序（最新優先），支持狀態篩選 |
| B | 列出所有訂單，支持狀態、日期範圍篩選與訂單編號搜尋 |
| C | 列出所有訂單，按狀態和日期分類顯示，支持分頁 |
| D | 簡單列出所有訂單，無篩選功能 |

# 影響範圍

- 查詢訂票記錄功能的用戶體驗
- 資料庫查詢設計與效能
- 前端過濾與搜尋功能

# 優先級

Low
- Low：不影響核心訂票邏輯，屬於 UX 增強功能

---
# 解決記錄

- **回答**：D - 簡單列出所有訂單，無篩選功能
- **更新的規格檔**：無需修改（與第 3、5 題決策一致）
- **變更內容**：
  - 決策：查詢訂票記錄功能不支援篩選、排序、搜尋、分頁等功能
  - 使用者查詢：簡單列出該使用者的所有訂單
  - 管理者查詢：簡單列出所有使用者的訂單
  - 不支援按狀態篩選、日期篩選、訂單編號搜尋、分頁顯示等功能
  - 此決策與第 3 題「不實作訂單篩選系統」、第 5 題「不支援條件查詢」一致
