# 釐清問題

刪除場次後，若該日期已無任何場次，UI 上如何顯示該日期？

# 定位

Feature：刪除場次.feature

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 保留空白日期：UI 仍顯示該日期的時刻表（無場次狀態），可繼續新增場次 |
| B | 自動隱藏：若日期無場次，該日期在時刻表 UI 中不顯示 |
| C | 顯示提示：顯示「尚無場次」提示，並提供新增場次入口 |
| Short | 提供其他簡短答案（<=5 字）|

# 影響範圍

- 時刻表 UI 設計
- DailySchedule 查詢與顯示邏輯
- 刪除場次後的 UI 狀態更新

# 優先級

Low
- 屬於 UI 細節設計，不影響核心功能

---
# 解決記錄

- **回答**：A - 保留空白日期：UI 仍顯示該日期（日曆上顯示灰色點代表 Draft 狀態），管理者可繼續新增場次
- **更新的規格檔**：無需更新（與之前「刪除場次後保留 DailySchedule」的決定一致）
- **變更內容**：確認無場次的日期仍顯示在日曆上（灰色點），與 DailySchedule 保留的行為一致
