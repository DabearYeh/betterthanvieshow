# 釐清問題

查詢訂票記錄功能中，可以透過哪些條件查詢？

# 定位

Feature：查詢訂票記錄功能
Rule：可以查詢所有訂票記錄

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 依場次查詢（影廳、日期、時間） |
| B | 依訂單狀態查詢 |
| C | 依電影查詢 |
| D | 以上皆可，支援多條件組合查詢 |
| Short | 提供其他簡短答案（<=5 字）|

# 影響範圍

- 查詢訂票記錄功能的規則與介面設計
- Order 實體的查詢索引設計
- 查詢功能的複雜度

# 優先級

Medium
- 影響查詢功能的使用性
- 影響後台管理的便利性

---
# 解決記錄

- **回答**：不支援條件查詢
- **更新的規格檔**：無需修改（與第 3 題決策一致）
- **變更內容**：
  - 決策：查詢訂票記錄功能不支援條件篩選
  - 使用者查詢：顯示該使用者的所有訂票記錄（無篩選）
  - 管理者查詢：顯示所有使用者的訂票記錄（無篩選）
  - 不支援依場次、訂單狀態、電影等條件篩選
  - 此決策與第 3 題「不實作訂單篩選系統」一致
