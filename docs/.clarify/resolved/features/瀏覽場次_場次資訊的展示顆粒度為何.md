# 瀏覽場次_場次資訊的展示顆粒度為何

瀏覽場次功能該展示哪些資訊？例如是否顯示剩餘座位數、票價、影廳座位圖等？

# 定義

Feature：瀏覽場次 > Rule: 可以查看場次時段資訊、影廳名稱

# 多選項

| 選項 | 描述 |
|--------|-------------|
| A | 最少資訊：場次時間、影廳名稱、票價 |
| B | 基本資訊：時間 + 票價 + 剩餘座位數 |
| C | 豐富資訊：時間 + 座位配置預覽 + 座位可選狀態 |
| D | 完整資訊：時間 + 所有資訊（含座位圖） |
| Short | 其他展示欄位清單 |

# 影響範圍

- 場次列表 API 響應的結構設計
- 前台展示的 UI 設計
- 系統效能（查詢場次時的資料量）

# 優先級

Low

---

**說明**：規格中未定義場次瀏覽的資訊細節,這會影響使用者體驗與 API 設計。

---

# 解決記錄

- **決議**：C  豐富資訊，列出時間（含開始時間，如 12:30、14:30 等）
- **參考設計稿**：如參考圖，顯示場次時間與剩餘座位
- **更新規格檔案**：spec/features/瀏覽場次.feature
