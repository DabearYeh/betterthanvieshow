# 密碼驗證規則 - 快速參考

## 新規則（2026-01-06 起）

### 最低要求
```
至少 6 字元
只能包含英文字母（a-z, A-Z）和數字（0-9）
```

### 正則表達式
```regex
^[a-zA-Z0-9]{6,}$
```

### 有效範例
| 密碼 | 說明 |
|------|------|
| `123456` | ✅ 純數字 |
| `password` | ✅ 純英文 |
| `pass12` | ✅ 英數混合 |
| `ABC123` | ✅ 大寫+數字 |

### 無效範例
| 密碼 | 原因 |
|------|------|
| `12345` | ❌ 少於 6 字元 |
| `pass@12` | ❌ 包含特殊符號 |
| `密碼123` | ❌ 包含中文 |
| `pass 12` | ❌ 包含空格 |

## 舊規則（2026-01-06 前）

### 最低要求
```
至少 8 字元
必須包含大寫字母、小寫字母和數字
```

### 正則表達式
```regex
^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).{8,}$
```

## API 測試

### 註冊 API
```bash
POST /api/auth/register
```

**有效請求**：
```json
{
  "name": "測試用戶",
  "email": "test@example.com",
  "password": "pass12"
}
```

**無效請求（密碼太短）**：
```json
{
  "name": "測試用戶",
  "email": "test@example.com",
  "password": "pass1"
}
```

**預期錯誤**：
```json
{
  "success": false,
  "message": "驗證失敗",
  "data": {
    "Password": ["密碼至少需 6 字元，且只能包含英文字母與數字"]
  }
}
```

## 常見問題

### Q: 舊帳號還能登入嗎？
**A:** 可以！登入驗證基於雜湊值比對，不檢查原始密碼格式。

### Q: 為什麼改簡單了？
**A:** 降低註冊門檻，提升使用者體驗，同時仍保有基本的安全性。

### Q: 可以用特殊符號嗎？
**A:** 不可以。只允許英文字母和數字。

### Q: 中文可以嗎？
**A:** 不可以。只允許英文字母和數字。
